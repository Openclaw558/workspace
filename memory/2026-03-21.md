### Root Cause Temuan
- **Bukan masalah network** — cemetery data preloaded tanpa API call
- **Angular CDK rendering timing**: `getByRole('combobox', { name: 'Cemeteries' })` kadang timeout karena accessible name belum attached ke accessibility tree
- **Kritis**: Pressing `Escape` untuk dismiss dropdown juga menutup entire dialog (bukan cuma dropdown) — bug di retry logic lama
- Angular generate dynamic IDs setiap dialog dibuka (`mat-select-2`, `mat-select-12`, `mat-form-field-label-3`, dll.)

### Fix yang sudah diapply ke `advanceSearch.steps.ts`
1. Step `I click Advanced search button without login`: Tambah polling loop (max 15s) untuk verify combobox accessible name "Cemeteries" sudah attached ke accessibility tree sebelum proceed
2. Step `I select cemetery`: 
   - Ganti `getByRole('combobox', { name: 'Cemeteries' })` → `.advanced-search-form mat-select:first-of-type` (hindari aria-labelledby dynamic ID issue)
   - Ganti `Escape` saat retry → `page.mouse.click(10, 10)` untuk dismiss dropdown tanpa close dialog
   - Tambah check: jika form tidak sengaja tertutup, reopen dan retry

Fix ini sudah dipatch tapi masih ada run yang fail — proses debugging masih ongoing saat compaction.
